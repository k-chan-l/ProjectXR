# ProjectXR

## 프로젝트 개요

XR 기술의 발전 가능성에 주목하여, 사용자가 건물 내부에서 원하는 위치까지 이동할 수 있도록 안내하는 XR 기반 길찾기 시스템을 개발하는 프로젝트입니다.

이 프로젝트는 건물 도면 정보를 바탕으로 주요 지점을 노드로 변환하고, 노드 간 연결 정보를 활용해 목적지까지의 경로를 탐색하는 것을 목표로 합니다.

실제 건물과 XR 장비를 활용한 테스트는 장비와 환경의 제약이 있기 때문에, Unreal Engine을 이용해 가상환경을 구성하고 실제 환경과 유사한 방식으로 시스템을 검증하고 있습니다.

## 프로젝트 구성

```
Unreal(Virtual Env) <--> Backend(Python)
```

Unreal Engine에서 생성되는 이미지 정보를 Python Backend로 전달하고, Backend는 도면 기반 노드 정보와 이미지 특징점을 활용해 현재 위치에서 목적지까지의 길찾기 정보를 제공합니다.

도면의 주요 길목을 Navigation Node로 만들고, 노드 간 연결 정보를 Edge로 기록합니다. 이후 Edge 정보를 바탕으로 Dijkstra 알고리즘을 적용하여 최적 경로를 탐색합니다.

탐색된 경로를 기준으로 사용자의 이동 방향을 안내하며, 현재 위치에서 다음 노드 방향을 판단하기 위해 이미지 특징점 기반 위치 인식 구조를 적용할 예정입니다.

## 실제와의 차이

실제 환경의 경우 구성요소

```
Env --> XR Glass(input) --> AI Backend --> XR Glass(output)
```

현재 프로젝트는 실제 XR 장비와 테스트 가능한 건물 환경이 부족하기 때문에 Unreal Engine 기반 가상환경을 사용합니다.

1. 사용 가능한 XR 장비의 부재로 Unreal Engine 가상환경을 사용
2. 실제 건물 도면 대신 가상 건물의 층별 이미지를 캡처하여 도면으로 활용
3. 최적의 노드를 완전히 자동으로 찾기 어려워 일부 노드 생성 과정은 수작업으로 보정
4. 노드 시각화를 위해 XR 영역이 아닌 Unreal 환경 내부에 Node Actor를 생성
5. 사용자가 최적 경로를 벗어나는 경우는 현재 단계에서 고려하지 않음
6. AI 기반 영상 인식 대신 경량 특징점 추출 방식을 우선 적용

## 현재 구현 완료

- FastAPI 기반 Unreal-Python 연동 구조 구축
- Unreal 환경에서 도면 캡처 파이프라인 구성
- Python 기반 2D 도면 Navigation Node 추출 기능 구현
- 도면 좌표와 Unreal 좌표 변환
- 추출된 Node를 Unreal 환경에 Actor로 생성하기 위한 파일 생성

## 개발 예정

- Node 간 Edge 정보 기반 경로 탐색
- Dijkstra 알고리즘 기반 목적지 경로 탐색
- 자동 캡처 지점 생성 및 이미지 저장
- ORB 특징점 기반 위치 인식
- 방향 기반 후보 노드 제한 구조 적용
- Unreal 환경 내 경로 시각화 UI 구현
